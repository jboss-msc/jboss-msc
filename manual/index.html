1. About

JBoss Modular Service Container (JBoss MSC) is a lightweight highly concurrent dependency injection container for Java. It is developed under the LGPL 2.1 license. The latest API documentation is available here.

2. Basic Concepts

JBoss MSC is built on three key abstractions.

2.1 Service

Service must be installed into service container to become functional. Every service has two lifecycle phases - start phase and stop phase. Services can start and stop either synchronously or asynchronously. Furthermore every service has:

  • identity - used for service's associated service controller retrieval from service container
  • dependencies - service may require values provided by other services
  • dependents - service may provide values required by other services
  • mode - defines service's starting policy
  • state - current state of service

2.2 Service Controller

Every service successfully installed into service container has associated service controller with it. Service controllers are useful for adding / removing lifecycle listeners to / from service at runtime, they can be used to change service mode at runtime (this includes also scheduling service for removal) or to retrieve some useful information about service such as its name, mode, state, unavailable dependencies and so on.

2.3 Service Container

Every successfully installed service is stored in service container. Service container provides information about available services and provides accessors for retrieving service controllers associated with installed services. Service container also provides shutdown mechanism to stop it.
